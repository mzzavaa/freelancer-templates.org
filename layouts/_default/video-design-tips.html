Video Design Tips

Make videos that get watched - not skipped

Research-backed principles from broadcast design, data visualization, and cognitive science. Applied to freelance video presentations at 1280x720.

Typography Layout Color Animation
01

Typography - size for the back row

Every text element must be readable when the video plays at 360p in an app sidebar. That means much bigger than you think.

Rule

Minimum body text: 32px at 1280x720

Broadcast standard requires text to be at least 2% of frame height. At 720px that is 14px - but that assumes a 40-inch TV at 2 meters. For web video watched on a laptop, scale up to 32px minimum for any readable text.

Scaled from: BBC/EBU broadcast spec (1920x1080 minimum 40px body text)
Rule

Maximum 4 levels of hierarchy per composition

Hero, title, body, caption. Use each level for exactly one purpose. Never use font size to create emphasis inside a single level - use weight or color instead.

Source: Bringhurst, The Elements of Typographic Style

02

Signal to noise - ruthless reduction

Edward Tufte's core principle: maximize the ratio of data to ink. Every element that does not add meaning reduces it.

One idea per scene

If a viewer needs to read two things at once, they read neither. Each scene should answer exactly one question. If you have two points, use two scenes with a cut.

  • One headline per frame - not a list of bullets
  • One number per stat card - not a dashboard
  • One action in the CTA - not "choose one of these"

Remove decoration, keep structure

Background textures, gradient overlays, drop shadows, and decorative borders are noise. Grid lines on charts are signal. Safe zones and consistent margins are signal. Animated blobs are noise.

  • Background grids: use at 4-8% opacity maximum
  • Drop shadows: only when showing depth is the point
  • Borders on cards: 1px at 20-30% opacity

Negative space is content

A frame with 40% empty space reads as confident. A frame with 90% text reads as a document. For 1280x720, leave at least 80px safe zones on all sides and never fill more than 65% of the frame with content.

Consistent margins = visual trust

Use the same padding value across every scene in a composition. Random-feeling spacing signals amateur work. This project uses PADDING=80 left/right and TOP_SAFE=48 - always use these constants, never hardcode.

03

Color - contrast and purpose

Color has one job in a video: direct attention. Every color that is not directing attention is competing with the ones that are.

Rule

4.5:1 contrast ratio minimum for all text

WCAG AA requires 4.5:1 for body text. This is also the threshold for text to read cleanly when video is compressed (H.264 compression amplifies low-contrast artifacts).

WCAG 2.1 Level AA / YouTube compression artifacts study, 2019
Rule

One accent color, one supporting color

Your accent is the "look here" color - used only for the most important element per scene. Supporting color is for secondary labels. Background and text are neutrals. Four colors maximum on screen at once.

Rule

Never use color alone to convey meaning

8% of men have color blindness. Use shape + color for status indicators, labels + color for categories, and icon + color for actions. A red warning badge must also say "Error" - the red alone is not enough.

04

Animation - purpose, not decoration

Every animation should direct focus, show a relationship, or give the viewer time to process information. If it does none of these, remove it.

Entrance timing: 0.3-0.6 seconds

At 30fps: 9-18 frames. Faster than 9 frames (0.3s) and elements "pop in" - viewers miss the movement. Slower than 18 frames (0.6s) and viewers feel like they are waiting. This project's DUR.fast=9 and DUR.normal=18 hit these exactly.

Stagger: 15-25 frames between items

When animating a list, stagger each item by half a second maximum. Longer staggers make a 4-item list feel like a 10-second video about nothing. Use staggerDelay() from animations.ts with 15-25 frame offsets.

Spring physics feel more natural than easing

CSS ease-out is fine for simple fades. For translations and scales, spring physics (mass, stiffness, damping) feel physically real because they model how objects actually decelerate. This is why Remotion's useSpring is better than interpolate for entrances.

Idle frames = breathing room

After all items have animated in, leave 30-60 idle frames (1-2 seconds) before the composition ends. This is when viewers actually read and retain the content. Cutting to black immediately after the last animation = zero retention.

05

Layout - visual hierarchy drives the eye

The eye enters a frame at the top-left and scans in a Z-pattern. Design your layout to match that path.

F-pattern for data-heavy layouts

The strongest position is top-left. Primary headline here. Secondary information goes on the second horizontal "stroke" of the F. Supporting details and stats fill the vertical "stem". Use BackgroundGrid to reinforce this structure visually.

Cards must be content-sized, not frame-filling

Never use flex:1 or align-stretch on card grids. Cards should be as tall as their content plus consistent padding. A 32px label in a 120px card is a layout error. Use alignContent:"start" on grid containers so rows wrap to content height.

Grid columns: 2-3 maximum

A 4-column grid at 1280px gives each column 280px after margins. At TYPE.body (32px), you can fit about 8-10 words per line. That is correct. Never go to 5+ columns - text wraps to 2-3 words per line which is unreadable.
