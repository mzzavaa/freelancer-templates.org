        title: "The Explainer Formula"
        body: "Effective explainers follow a proven structure: Problem (what's confusing?), Solution (the concept explained simply), Example (real-world application), and Summary (key takeaway). This formula works for any topic complexity."
        source: "Instructional Design Principles"

        title: "Simplification Without Dumbing Down"

  - title: "Step-by-Step Guides"
    description: "Create clear, actionable step-by-step guides that walk learners through processes with confidence and minimal confusion."
    columns: 2
    cards:
      - type: "rule"
        title: "Step Numbering Best Practices"
        body: "Always number your steps clearly and visibly. Use large, contrasting numbers that stay on screen during each step. Limit to 5-7 steps per video—if you need more, break into multiple videos."
        source: "Instructional Design Guidelines"

      - type: "example"
        title: "Software Tutorial Steps"
        body: "A productivity app creates onboarding tutorials using the Platform Overview template with clear step progression."
        list:
          - "Step 1: Create your account"
          - "Step 2: Set up your workspace"
          - "Step 3: Invite team members"
          - "Step 4: Create your first project"
          - "Step 5: Customize your dashboard"

      - type: "tip"
        title: "Visual Step Transitions"
        body: "Use clear visual transitions between steps. The step number should animate in, the previous content should exit, and the new content should enter with purpose. This signals progression and helps viewers track where they are."

      - type: "example"
        title: "Step-by-Step Elements"
        body: "Visual components that make step-by-step guides easy to follow and remember."

      - type: "rule"
        title: "Action-Oriented Language"
        body: "Start each step with a strong action verb: Click, Type, Select, Drag, Open, Navigate. Avoid passive language. \"Click the blue button\" is clearer than \"The blue button should be clicked.\""
        source: "Technical Writing Standards"

      - type: "tip"
        title: "Error Prevention"
        body: "Anticipate common mistakes and address them proactively. Add callouts like \"Make sure you're logged in first\" or \"Don't click Submit yet.\" Preventing errors is better than troubleshooting them."

  - title: "Retention Techniques"
    description: "Apply proven learning science techniques to create educational videos that stick in memory and drive real skill development."
    columns: 3
    cards:
      - type: "rule"
        title: "Spaced Repetition"
        body: "Reinforce key concepts by revisiting them at increasing intervals. Mention important points at the beginning, middle, and end of videos. Create recap videos that review previous lessons before introducing new material."
        source: "Learning Science Research"

      - type: "example"
        title: "Retention-Focused Structure"
        body: "A language learning app structures lessons for maximum retention using the Animated List template."
        list:
          - "Preview: What you'll learn today"
          - "Review: Quick recap of last lesson"
          - "New content: Today's material"
          - "Practice: Interactive examples"
          - "Summary: Key takeaways repeated"

      - type: "tip"
        title: "Active Recall Prompts"
        body: "Pause and ask viewers questions before revealing answers. \"What do you think happens next?\" or \"Can you guess the solution?\" Active recall strengthens memory far more than passive watching."

        source: "Cognitive Psychology"
